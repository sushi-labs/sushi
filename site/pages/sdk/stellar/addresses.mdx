# Addresses

The Stellar module provides typed account addresses, contract addresses, transaction hashes, and normalization helpers for the Stellar network.

## StellarAccountAddress

`StellarAccountAddress` represents a classic Stellar account public key. These are [StrKey](https://stellar.org/protocol/sep-23) encoded Ed25519 public keys and always start with `G`.

```ts twoslash
import type { StellarAccountAddress } from 'sushi/stellar'

const address: StellarAccountAddress =
  'GAAZI4TCR3TY5OJHCTJC2A4QSY6CJWJH5IAJTGKIN2ER7LBNVKOCCWN7'
```

## StellarContractAddress

`StellarContractAddress` represents a Soroban contract address. These are also StrKey-encoded and always start with `C`.

```ts twoslash
import type { StellarContractAddress } from 'sushi/stellar'

const contractAddress: StellarContractAddress =
  'CCRSMJDITH3VK5QOGYCVZDAKIY5GL3RCG4TCVLIAVB662IW2V5KJGZGF'
```

## StellarAddress

`StellarAddress` is the union used when an API may accept either an account or contract address.

```ts twoslash
import type { StellarAddress } from 'sushi/stellar'

const address: StellarAddress =
  'CCRSMJDITH3VK5QOGYCVZDAKIY5GL3RCG4TCVLIAVB662IW2V5KJGZGF'
```

## StellarTxHash

`StellarTxHash` represents a Stellar transaction hash:

```ts twoslash
import type { StellarTxHash } from 'sushi/stellar'

const txHash: StellarTxHash =
  '3389e9f0f1a65f19736cacf544c2e825313e8447f569233bb8db39aa607c8889'
```

## Validation

Use the explicit validator that matches the address kind you expect:

```ts twoslash
import {
  isStellarAccountAddress,
  isStellarAddress,
  isStellarContractAddress,
} from 'sushi/stellar'

isStellarAccountAddress(
  'GAAZI4TCR3TY5OJHCTJC2A4QSY6CJWJH5IAJTGKIN2ER7LBNVKOCCWN7',
)
// true

isStellarContractAddress(
  'CCRSMJDITH3VK5QOGYCVZDAKIY5GL3RCG4TCVLIAVB662IW2V5KJGZGF',
)
// true

isStellarAddress('GAAZI4TCR3TY5')
// false
```

The validation regexes are:

```
/^G[A-Z2-7]{55}$/
/^C[A-Z2-7]{55}$/
```

## Normalization

Use `normalizeStellarAddress` for both account and contract addresses:

```ts twoslash
import { normalizeStellarAddress } from 'sushi/stellar'
import type {
  StellarAccountAddress,
  StellarContractAddress,
} from 'sushi/stellar'

const normalizedAccount = normalizeStellarAddress(
  'gaazi4tcr3ty5ojhctjc2a4qsy6cjwjh5iajtgkin2er7lbnvkoccwn7' as StellarAccountAddress,
)

const normalizedContract = normalizeStellarAddress(
  'ccrsmjdith3vk5qogycvzdakiy5gl3rcg4tcvliavb662iw2v5kjgzgf' as StellarContractAddress,
)

const normalizedGeneric = normalizeStellarAddress(
  'ccrsmjdith3vk5qogycvzdakiy5gl3rcg4tcvliavb662iw2v5kjgzgf' as StellarContractAddress,
)
```

When working with the generic layer, `normalizeAddress` from `sushi/generic` automatically delegates to `normalizeStellarAddress` for Stellar chain IDs.

## Native Address

The Stellar native asset (XLM) is represented by a special account-style constant address:

```ts twoslash
import { stellarNativeAddress } from 'sushi/stellar'

stellarNativeAddress
// 'GAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA'
```

Similarly, the generic `nativeAddress` from `sushi/generic` returns this constant when given a Stellar chain ID.

## Generic Type Integration

Stellar address and transaction hash types are integrated into the generic conditional types:

```ts twoslash
import type { AddressFor, TxHashFor } from 'sushi/generic'
import type { StellarChainId } from 'sushi/stellar'

type Addr = AddressFor<StellarChainId> // StellarAddress
type Hash = TxHashFor<StellarChainId> // StellarTxHash
```
